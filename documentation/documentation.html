Idee & Konzept

Idee

Vor allem in Diskotheken und auf Veranstaltungen werden oft Licht- und Sound-Effekte eingesetzt um die Besucher zu unterhalten. Leider werden diese Effekte meist nur von einzelnen Personen oder programmierten Automatiken gesteuert, die Besucher können diese also nur passiv erleben.

Meine Idee ist es dem Besucher die Möglichkeit zu bieten selbst aktiv zu werden und unkompliziert die Kontrolle über einzelne Effekte zu übernehmen. Dazu soll ein möglichst intuitiv bedienbares Web-Interface erstellt werden, welches über jedes Endgerät mit Internetverbindung und modernem Browser aufgerufen und bedient werden kann. Zu diesen Geräten zählen zum Beispiel Smartphones, Tablet-Computer oder auch Touch-Tische. So kann jeder Besucher einer Veranstaltung die Kontrolle über beispielsweise ein Licht übernehmen.

Die steuerbaren Effekte werden vorher von dem Veranstalter bzw. einer zuständigen Person konfiguriert. Dabei werden zum Beispiel mögliche Farben, Filter und Bewegungen eingestellt und falls nötig eingeschränkt.

Die fertige Applikation soll kostenfrei und offen zur Verfügung gestellt werden (Open Source). Für Infos, Download und Installationsanweisungen kann eine kleine Website erstellt werden sowie eine Dokumentation des Codes zur einfachen Erweiterung durch Dritte.

Konzept

Die zu erstellenden Komponenten der Applikation lassen sich in die drei Bereiche Client (Sender), Server (Empfänger) und Netzwerk (Kommunikationsweg) aufteilen:

Client (Sender)

Die Aufgabe dieses Clients wird es sein die Steuerbefehle des Users zu erfassen, aufzubereiten und mit möglichst wenig Verzögerung an den Server zu senden. Dabei müssen verschiedene Eingabemethoden wie Touch, Maus oder Tastatur berücksichtigt werden. Die Applikation soll mit der aktuellen Technologie von HTML, CSS und Javascript umgesetzt werden um auf jedem Endgerät mit aktuellem Browser zu funktionieren und damit kein extra Programm auf dem Gerät installiert werden muss. Da es sich bei den meisten Client-Geräten um Smartphones und Tablet-Computer handeln wird sollte viel Wert auf eine Optimierung für diese Geräte gelegt werden. Um eine möglichst geringe Latenz zu erreichen ist es sinnvoll Web-Sockets zu verwenden.

Server (Empfänger)

Der Server wird die vom Client empfangenen Steuerbefehle je nach Konfiguration validieren, weiterverarbeiten und letztendlich auf ein eingestelltes Protokoll übersetzen um das entsprechende Effektgerät bzw. die entsprechende Software anzusprechen. Als Schnittstelle zu diesen bieten sich das OSC, das MIDI oder das DMX Protokoll an, welche zur Ansteuerung von Lichteffekten und Musikinstrumenten weit verbreitet sind und deshalb von den meisten Geräten und Programmen zur Steuerung von Effekten unterstützt werden. Somit lässt sich der Server einfach in einen vorhandenen Aufbau integrieren. Der Server wird sich außerdem um die Ausspielung der Client-Applikations-Daten kümmern und die Verbindungen der Clients über Websockets verwalten.

Netzwerk (Kommunikationsweg)

Um speziell für die mobilen Client-Geräte eine möglichst geringe Latenzzeit zu erreichen und die Verwendung des Systems an jedem Ort zu ermöglichen sollte für die Applikation vor Ort ein eigenes Netzwerk eingerichtet werden. Die Steuerbefehle legen über eine mobile Datenverbindung eine wesentlich längere und somit auch langsamere Strecke bis zum Server zurück als bei einer direkten Verbindung über W-LAN. Dazu kommt, dass auf gut besuchten Veranstaltungen durch die vielen Geräte der Besucher das sowohl mobile Funknetz und als auch die Internetverbindung stark ausgelastet sind und sich für die Echtzeitsteuerung nicht eignen. Auch schirmen die Räumlichkeiten mancher Gebäude das Signal zu stark ab, so dass eine stabile mobile Verbindung nicht immer gewährleistet ist. Durch die Einrichtung eines öffentlichen W-LAN Zugangs zum Server-Netzwerk können diese Probleme umgangen und die Latenzzeit verringert werden. Zugang zum Internet sollte über dieses Netzwerk nicht geboten werden, deshalb muss ein eigener DNS Service eingerichtet werden.

Arbeitsweise

Die aktuellste Version der Applikation wird regelmäßig zur Versionsverwaltung in ein GitHub Repository gepusht. Im Code selbst werden Kommentare im “JSDoc” Stil geschrieben um mit Hilfe des gleichnamigen Programms eine Code-Dokumentation generieren zu können. Die Dokumentation der Arbeit selbst wird in Form einer Webseite erstellt und mit Hilfe von «Prince» als druckbares PDF exportiert. “Prince” ist eine Software welche Druckbare PDF- Dateien aus HTML Dokumenten generiert.
